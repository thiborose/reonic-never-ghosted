# 03 Installer Sales Process

Status: completed synthesis on 2026-06-20.

## Core Process

```text
offer_sent_or_opened
  -> diagnose blocker and missing data
  -> check consent and claim safety
  -> choose blocker-resolving action
  -> prepare proof, script, proposal change, or document request
  -> schedule against owner capacity and travel
  -> execute action
  -> debrief
  -> update strategy, next commitment, and outcome
```

## Daily Priority Queue

| Priority | Trigger | Recommended response | Evidence |
| --- | --- | --- | --- |
| 1 | Customer asked a question or replied | Answer through eligible channel; schedule call if complex | A6, A3 |
| 2 | Installer promised a next step | Complete or send status update | A6, A2 |
| 3 | Real deadline | Use factual quote validity, subsidy/tariff window, or customer-stated deadline | A4, A5 |
| 4 | Proof work blocks claim | Request data, refresh market fact, revise proposal | A3, A4, A6 |
| 5 | High-value/high-readiness blocker | Schedule call/visit only if justified | A6 |
| 6 | Stale no-response | One value-add touch or pause | A5, A6 |

## Capacity Rules For Small/Mid Installers

- Do not default to in-person visits. Use visits when physical inspection, roof/property risk, heat-load/building uncertainty, high trust risk, high lead value, or route cluster justifies travel.
- Prefer virtual consultation, call, video, proposal revision, or document request when the blocker is explanation or missing information.
- Batch quick follow-ups and document requests into admin blocks.
- Reserve owner/senior time for high-value, high-readiness, trust-sensitive, or technically complex leads.
- Every visit action needs a purpose: "what will this visit unlock?"

## Required Sales Objects

| Object | Minimum fields |
| --- | --- |
| Action | type, blocker, reason, owner, channel, duration, preparation assets, success criteria, compliance checks |
| Calendar | owner availability, conflicts, working hours, travel time, start/end location, route cluster, manual override reason |
| Debrief | completed/skipped, customer response, customer wording, resolved objection, remaining objection, sentiment, next commitment, outcome label |
| Outcome | signed, still evaluating, needs data, competitor risk, lost price, lost trust, paused, not interested, no response |

## Unvalidated Process Claims

- Exact cadence by channel is not validated for German PV/heat-pump quote-stage sales.
- Offer opens/views are not proven predictors of signing and require tracking consent.
- In-person visit thresholds need real installer calendar and outcome data.
- Debrief must be short enough for owner-led installers; test it.

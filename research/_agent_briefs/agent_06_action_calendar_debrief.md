# Agent 6 Brief: Action, Calendar, And Debrief

## Mission

Define how the assistant recommends, prepares, schedules, and learns from actions beyond message drafting.

## Work Folder

Use only: `research/agents/agent_06_action_calendar_debrief/`

## Core Questions

- When should the next best action be a call, email, SMS, WhatsApp, personalized video, proposal revision, document request, virtual consultation, in-person visit, pause, or escalation?
- When is an in-person visit worth the time and travel cost for a small German installer?
- How should the assistant prioritize leads when the installer has many active leads and limited calendar capacity?
- What scheduling data should the mock PoC include?
- What debrief questions should be asked after each action type?
- How should strategy change after positive, negative, neutral, or ambiguous debriefs?

## Required Outputs

- Action and debrief records using `_templates/action_debrief_record.md`.
- Source records in `sources.md`.
- Evidence items in `evidence.md`.
- Findings in `findings.md`.
- Handoff in `handoff.md`.
- Open questions in `open_questions.md`.

## Research Rules

- Treat installer time as a scarce resource.
- Do not assume in-person visits are always best.
- Tie every action recommendation to persona, objection, lead value, urgency, compliance, and capacity.
